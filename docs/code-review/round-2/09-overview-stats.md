## 九、关键建议

### 对技术负责人的建议

1. **安全第一**：当前最紧迫的问题是 Session 伪造和硬编码密钥，这两项可让攻击者无需认证即可获得管理员权限，应立即组织修复。

2. **Lint 工具统一**：当前 Biome vs ESLint 的双轨问题是关键架构决策点。`no-prisma-enum-in-api-property` 规则非常重要不能失效，需尽快确定统一方案。

3. **类型安全逐步恢复**：后端 `strictNullChecks: false` 是历史债务的集中体现。建议逐模块开启（先从 conversion、fonts 等小模块开始），配合补充单元测试。

4. **CAD 引擎黑盒盲区**：11号报告中有10项须与 mxcad-app 厂商确认。建议建立定期的三方沟通机制，减少黑盒盲区。

5. **测试投资回报**：当前集成测试质量良好（perm Auth、SVN），应以此为模板推广。但同时应避免 `file-system.service.spec.ts` 式的纯委托验证测试。
