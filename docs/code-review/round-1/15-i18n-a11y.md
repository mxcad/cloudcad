## 1. 国际化支持 (i18n)

### 1.1 整体情况：无国际化方案

**严重程度：高**

项目**完全没有国际化框架和基础设施**。搜索结果：
- 无 i18next、react-intl、formatjs 等任何 i18n 库
- 无 locale JSON 翻译文件（node_modules 除外）
- 无 `navigator.language` 浏览器语言检测
- 无 `Accept-Language` header 后端解析
- 无 Cookie/URL 参数语言切换

**影响范围**：整个应用仅支持中文，无法服务多语言用户。

## 4. 可访问性 — ARIA 属性

### 问题 ARIA-01：自定义 Modal 缺少关键 ARIA 属性

- **文件**: `packages/frontend/src/components/ui/Modal.tsx:47-181`
- **严重程度**: 高
- **描述**: 自定义 Modal 组件（非 Radix Dialog）缺少以下关键属性：
  - 无 `role="dialog"` 声明
  - 无 `aria-modal="true"` 标记
  - 无 `aria-labelledby` 关联标题
  - 无 `aria-describedby` 关联内容描述
  - 无焦点管理（打开时未聚焦模态框，关闭时未归还焦点）
  - 无 Escape 键处理
- **修复建议**: 
  1. 外层 div 添加 `role="dialog" aria-modal="true"`
  2. 标题 h3 添加 id，外层 div 添加 `aria-labelledby` 关联
  3. 实现焦点捕获：打开时聚焦第一个可聚焦元素，Tab 在模态框内循环
  4. 添加 Escape 键关闭支持
  5. 关闭后恢复打开前的 `document.activeElement` 焦点
- **架构建议**: 考虑用 Radix UI `@radix-ui/react-dialog` 替代自定义 Modal，Radix 自带完整的 a11y 实现
- **是否需用户确认**: 是 — 可能需要评估迁移到 Radix Dialog 的工作量

### 问题 ARIA-02：ConfirmDialog 完全缺少 ARIA 属性

- **文件**: `packages/frontend/src/components/ui/ConfirmDialog.tsx:15-117`

  - 无 `role="alertdialog"`

- **是否需用户确认**: 否

## 6. 可访问性 — 色彩对比度

### 问题 COLOR-01：主题色彩需验证 WCAG AA 对比度

- **文件**: `packages/frontend/src/styles/theme.css`
- **严重程度**: 中
- **描述**: 使用 CSS 变量系统定义色值，部分组合可能不满足 WCAG AA 标准：
  - 亮色主题 `--text-muted: #94a3b8` 在 `--bg-primary: #f8fafc` 上：对比度约 2.16:1 — **不满足 AA**（需 ≥4.5:1 正常文本，≥3:1 大文本）
  - 亮色主题 `--text-tertiary: #64748b` 在 `--bg-primary: #f8fafc` 上：对比度约 3.54:1 — 仅满足大文本 AA
  - 深色主题 `--text-muted: #5a6a7a` 在 `--bg-primary: #141619` 上：对比度约 3.58:1 — 仅满足大文本 AA
  - 深色主题 `--text-tertiary: #7a8a99` 在 `--bg-primary: #141619` 上：对比度约 4.28:1 — 不满足正常文本 AA
- **修复建议**: 
  - 使用 WebAIM Contrast Checker 或 axe DevTools 验证所有颜色组合
  - `text-muted` 色值在亮色主题可调整为 `#64748b`（slate-500，对比度 4.4:1）或 `#737373`
- **是否需用户确认**: 否

### 问题 COLOR-02：主题切换未考虑系统偏好

- **修复建议**: 初始加载时，如果 localStorage 无值，使用 `matchMedia('(prefers-color-scheme: dark)')` 作为默认值
- **是否需用户确认**: 否

## 7. 可访问性 — 减少动画支持

### 问题 MOTION-01：prefers-reduced-motion 已注释掉

- **文件**: `packages/frontend/src/styles/theme.css:297-318`
- **严重程度**: 中
- **描述**: `prefers-reduced-motion: no-preference` 的媒体查询代码块**已被完全注释掉**（为"防止主题切换闪烁"）。这导致动画对所有用户始终以全速播放，无视用户系统偏好设置。
- **修复建议**: 
  - 添加 `@media (prefers-reduced-motion: reduce)` 规则，禁用或减弱动画
  - 主题过渡动画可以在 `prefers-reduced-motion: no-preference` 下启用
  ```css
  @media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
      animation-duration: 0.01ms !important;
      transition-duration: 0.01ms !important;
    }
  }
  ```
  - 同时使用 Tailwind 的 `motion-safe:` / `motion-reduce:` 修饰符
- **是否需用户确认**: 否

### 问题 MOTION-02：多处内联动画未考虑 reduced-motion

- **文件**: 
  - `packages/frontend/src/components/ui/Modal.tsx:153-176`（内联 keyframe 动画）
  - `packages/frontend/src/components/InteractiveBackground.tsx:85-194`（canvas 动画循环）
- **严重程度**: 中
- **描述**: 
  - Modal 的 `modalFadeIn` 和 `modalScaleIn` 动画总是播放
  - InteractiveBackground 的 requestAnimationFrame 动画循环始终运行（即使 `prefers-reduced-motion: reduce`）
- **修复建议**: 
  - Modal 动画添加 `@media (prefers-reduced-motion: reduce)` CSS 禁用
  - InteractiveBackground 检测 `matchMedia('(prefers-reduced-motion: reduce)')` 后停止动画循环
- **是否需用户确认**: 否
