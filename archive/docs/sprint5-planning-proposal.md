# 冲刺五规划建议

> **项目名称:** CloudCAD 在线 CAD 协同平台  
> **报告时间:** 2026-05-03  
> **分支:** refactor/circular-deps

## 概述

本规划建议基于冲刺四的完成状态和审计发现的问题，提出冲刺五可重点推进的四个方向：
1. 性能深度优化
2. 测试覆盖率提升
3. 交互体验打磨
4. 文档体系完善

## 1. 性能深度优化

### 1.1 后端数据库优化

#### 问题清单
| 问题 | 优先级 | 位置 | 影响 |
|-----|-------|------|------|
| 50% 数据库查询缺少索引 | 高 | 审计报告 A.2.2 | 数据库性能 |
| 存在 N+1 查询问题 | 高 | 审计报告 A.2.3 | 查询性能 |
| Controller 直接操作 Prisma | 中 | 审计报告 A.2.1 | 架构规范性 |
| 缺少死锁重试机制 | 中 | 审计报告 A.2.5 | 可靠性 |
| 部分查询未分页 | 中 | 审计报告 A.2.4 | 性能 |

#### 优化计划

**P1 - 紧急（第一周）**

| 任务 | 目标 | 验收标准 |
|-----|------|---------|
| 为常用查询字段添加索引 | 为 `userId`、`parentId`、`type`、`status`、`deletedAt` 等字段添加索引 | 所有查询都走索引，无全表扫描 |
| 修复 N+1 查询问题 | 使用 `include`、`select` 或专门的查询方法 | 消除所有 N+1 查询 |
| 为关键写操作添加死锁重试 | 使用 `prisma.$transaction` + 重试机制 | 死锁率降为 0 |

**P2 - 重要（第二周）**

| 任务 | 目标 | 验收标准 |
|-----|------|---------|
| 所有列表查询强制分页 | 移除所有 `findMany()` 无分页调用 | 所有列表都有分页 |
| 重构 Controller 直接调用 Prisma | 通过 Service 层访问数据库 | 100% 通过 Service 访问 |
| 添加数据库查询性能监控 | 记录慢查询，设置告警阈值 | 有监控数据和告警 |

### 1.2 前端构建性能优化

#### 问题清单
| 问题 | 优先级 | 位置 | 影响 |
|-----|-------|------|------|
| 构建时间过长（约 3 分钟） | 中 | 审计报告 D.1.1 | 开发体验 |
| 初始加载时间过长（约 5-6 秒） | 中 | 审计报告 D.1.2 | 用户体验 |
| 未使用 Tree Shaking | 低 | 审计报告 D.1.4 | 包大小 |

#### 优化计划

**P1 - 紧急（第一周）**

| 任务 | 目标 | 验收标准 |
|-----|------|---------|
| 启用 Vite 预构建优化 | 配置 `optimizeDeps` | 开发模式启动时间 < 30 秒 |
| 实现路由懒加载 | 所有路由页面动态导入 | 初始加载时间 < 3 秒 |
| 配置构建缓存 | 使用缓存加速重复构建 | 二次构建时间 < 1 分钟 |

**P2 - 重要（第二周）**

| 任务 | 目标 | 验收标准 |
|-----|------|---------|
| 分析并优化第三方库 | 移除未使用的库，替换大库 | 打包体积减少 30% |
| 启用 Tree Shaking | 配置 `sideEffects` | 无死代码 |
| 添加 Bundle Analyzer | 集成 rollup-plugin-visualizer | 有可视化分析报告 |

## 风险和依赖

### 风险
1. **性能优化可能引入新 Bug**：需要充分测试
2. **测试覆盖率提升耗时较长**：需要合理安排优先级
3. **文档编写需要投入精力**：需要技术写作能力

### 依赖
1. 需要开发团队配合优化和测试
2. 需要产品团队确认交互设计
3. 需要运维团队配合监控和部署

## 总结

冲刺五规划建议基于冲刺四的完成状态和审计发现，重点推进四个方向：

1. **性能深度优化**：数据库优化、前端构建优化、运行时优化
2. **测试覆盖率提升**：后端测试、前端测试、集成测试
3. **交互体验打磨**：错误处理、表单验证、导航优化
4. **文档体系完善**：API 文档、架构文档、开发文档、用户文档

通过两周的冲刺，可以显著提升系统的性能、稳定性、可维护性和用户体验。
