# PC 端移动端适配计划

## 目标
除 CAD 编辑器外，PC 前端所有页面在手机浏览器中正常使用（不跳转 H5）。

## 现状分析
- Layout 组件已有基础移动端支持（侧边栏滑入/滑出、遮罩层、菜单按钮）
- Dashboard 已有响应式网格（`grid-cols-2 lg:grid-cols-4`）
- `isMobile()` 工具函数存在，但仅用于 CAD 路由跳转判断
- 移动端跳转逻辑：当前所有移动端访问 CAD 路由时跳转到 H5 应用
- 部分页面有 `@media (max-width: 640px/768px)` 样式但不系统

## 实施步骤

### 1. 修改移动端跳转逻辑
**文件**: `packages/frontend/src/App.tsx`

- 修改 `CADEditorRouteGuard` 中的 `isMobile()` 判断，仅对 CAD 编辑器路由生效
- 非 CAD 路由（登录、注册、仪表盘、文件管理等）移动端不再跳转
- 现有逻辑：`if (!isMobile())` → 跳过。改为：`if (!isMobile() || !isCADRoute)` → 跳过

### 2. 创建 `useIsMobile` Hook
**文件**: `packages/frontend/src/hooks/useIsMobile.ts`（新建）

- 基于 `window.matchMedia('(max-width: 768px)')` 响应式检测
- 返回 boolean，屏幕宽度变化时自动更新
- 供所有组件使用，替代硬编码的媒体查询判断

### 3. 优化 Layout 移动端交互
**文件**: `packages/frontend/src/components/Layout.tsx`

- 移动端侧边栏改为全屏覆盖（当前 `w-72` 在小屏仍占空间）
- 顶部导航栏移动端精简：隐藏时间显示（已有 `hidden md:flex`）
- 添加移动端底部导航栏（可选，提升导航效率）
- 侧边栏导航项在移动端增加触摸友好的点击区域（min-height: 44px）

## 验证方式
1. Chrome DevTools 切换到移动设备模拟器（iPhone SE / iPhone 14 / Android）
2. 逐页面检查：登录 → 仪表盘 → 项目管理 → 个人资料 → 管理页面
3. 确认 CAD 编辑器路由仍正常跳转到 H5
4. 确认所有页面在 320px ~ 768px 宽度下可正常使用
5. 运行 `pnpm type-check` 确认无类型错误
