# FloatingPopup 组件修复计划

## 问题分析

`packages/frontend_mobile/src/components/FloatingPopup.vue` 存在 3 个核心问题：

### 1. 双重拖拽横条
- `van-floating-panel` 自带内置拖拽条（vant 默认渲染）
- 组件又手动添加了 `.floating-popup-bar` div
- 导致显示**两个**拖拽条

### 2. 拖拽功能异常
- `van-popup` 的 style 绑定了 `height: ${currentHeight}px`
- `van-floating-panel` 内部也用 `v-model:height="currentHeight"` 管理高度
- 两者同时控制同一个高度值，形成循环依赖：
  - 拖拽 → panel 更新 `currentHeight` → popup 高度变化 → panel 容器变化 → 拖拽行为异常
- `van-popup` 不应该直接控制高度，应该让 `van-floating-panel` 独立管理

### 3. 拖拽条/标题显示逻辑
- 当前 bar 始终显示（即使有标题）
- 正确逻辑：有标题时显示标题，无标题时才显示拖拽条

## 修复方案

### 核心思路：移除 `van-floating-panel`，改用纯 `van-popup` + 手动拖拽

`van-floating-panel` 设计为独立组件，嵌套在 `van-popup` 内部会导致高度管理冲突。更好的方案是：

1. **仅使用 `van-popup`** 作为容器（提供遮罩、动画、teleport）
2. **手动实现拖拽逻辑**（拖拽条 + 锚点吸附）
3. **条件渲染**：有标题时显示标题栏，无标题时显示拖拽条

### 修改文件

`packages/frontend_mobile/src/components/FloatingPopup.vue`

### 实现细节

#### 模板结构
```
van-popup (position="bottom", 不绑定 height)
  └─ 拖拽区域
      ├─ 拖拽条 (仅在无标题时显示)
      ├─ 标题栏 (仅在有标题时显示)
      ├─ header-after 插槽
      ├─ 内容区
      └─ 底部插槽
```

#### 拖拽逻辑
1. 监听拖拽条/标题栏的 `touchstart`、`touchmove`、`touchend`
2. 根据手指位移计算新高度
3. 高度变化时检查是否接近锚点值，实现吸附效果
4. 在 `touchend` 时，动画过渡到最近的锚点

#### 锚点计算
- 保留现有 `realAnchors` 计算逻辑（`<=1` 视为百分比，`>1` 视为 px）
- 默认锚点 `[0.6, 0.9]` 即 60% 和 90% 屏幕高度

#### 关键代码变更

1. **移除** `van-floating-panel` 包裹层
2. **移除** `currentHeight` 对 popup style 的绑定（popup 应用固定高度样式由拖拽控制）
3. **添加** 触摸事件处理函数（`onDragStart`, `onDragMove`, `onDragEnd`）
4. **修改** 拖拽条渲染条件：`v-if="!title && !$slots.header"`
5. **保持** 所有现有 props/emits/slots 兼容

### 样式变更

- `.floating-popup-bar`：添加 `touch-action: none` 和 `cursor: grab` 以支持拖拽
- 移除 `van-floating-panel` 相关的 CSS 变量引用（如 `--van-floating-popup-border-radius`）

## 验证方式

1. `pnpm build` 确保编译通过
2. 在移动端预览中测试：
   - 有标题时：显示标题栏，无拖拽条
   - 无标题时：显示拖拽条，可拖拽
   - 拖拽到锚点附近时自动吸附
   - 点击遮罩正常关闭
   - 底部安全区正常
3. 检查所有使用 `FloatingPopup` 的组件（PdfOptionsPopup, CooperatePopup, SaveAsSheet 等）功能正常
