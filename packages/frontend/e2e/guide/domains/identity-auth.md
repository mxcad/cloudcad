# 身份权限域 — 测试策略

## 页面清单

| 页面 | 路由 | 权限要求 | 说明 |
|------|------|---------|------|
| 登录 | `/login` | 公开 | 账号密码登录、微信登录、手机登录 |
| 注册 | `/register` | 公开 | 用户注册、注册关闭提示 |
| 邮箱验证 | `/verify-email` | 公开 | 邮箱验证码确认 |
| 手机验证 | `/verify-phone` | 公开 | 手机验证码确认 |
| 忘记密码 | `/forgot-password` | 公开 | 密码重置请求 |
| 重置密码 | `/reset-password` | 公开 | 新密码设置 |
| 个人中心 | `/profile` | 需登录 | 个人信息、修改密码、手机/邮箱绑定、微信绑定、注销 |
| 项目成员管理 | 组件内弹窗 | 需 PROJECT_MEMBER_MANAGE | MembersModal、ProjectRolesModal |

## 核心工作流

### 登录注册流程
1. 访问受保护页面 → 重定向到 /login → 登录成功 → 跳转回原页面
2. 登录 → 登录失败（错误凭证/锁定/禁用） → 错误提示
3. 注册 → 填写表单 → 提交 → 邮箱验证 → 完成注册
4. 忘记密码 → 输入邮箱 → 收邮件 → 重置密码 → 登录
5. 登录后 token 过期 → 自动刷新 → 刷新失败 → 跳转登录

### 权限控制流程
1. 不同系统角色访问受保护页面 → 有权限/无权限（NoPermissionPage）
2. 不同项目角色操作文件 → 按钮显隐/操作被拒（Toast 提示）
3. 未登录访问保护页面 → 重定向登录 → 登录后恢复原始目标
4. 用户注销 → 清除所有会话 → 重定向 /login
5. 用户自助注销 → 确认弹窗 → 注销成功 → 30天内可恢复

### 成员管理流程
1. 添加成员 → 搜索用户 → 选择角色 → 确认添加 → Toast
2. 移除成员 → 确认弹窗 → 移除成功
3. 修改成员角色 → 下拉选择 → 确认变更
4. 所有权转让 → 选择新 OWNER → 输入密码确认 → 转让成功（当前用户降级）
5. 获取成员列表 → 按角色分组显示

## 交互覆盖要点

### 登录页
- 表单校验：空账号/空密码/格式错误提示
- 密码显隐切换
- 记住我 checkbox
- 微信登录按钮（扫码弹窗）
- 手机登录 tab 切换
- 注册链接跳转
- 忘记密码链接跳转
- 登录失败错误提示（toast/inline）
- 登录按钮 loading 状态
- 回车键提交

### 注册页
- 必填字段校验
- 用户名/邮箱格式校验
- 密码强度校验
- 确认密码一致性
- 验证码（如启用）
- 注册关闭状态（RegistrationClosed 组件）
- 提交 loading

### 个人中心
- 5 个 Tab：账户信息、修改密码、邮箱、手机、微信、注销
- 修改密码：旧密码 → 新密码 → 确认 → 提交
- 邮箱绑定：输入邮箱 → 验证码 → 确认
- 手机绑定：输入手机 → 验证码 → 确认
- 微信绑定：扫码 → 确认绑定
- 注销：确认弹窗 → 原因 → 提交

### MembersModal（项目成员管理）
- 成员列表 → 按角色排序
- 添加成员 → 搜索下拉 → 角色选择 → 确认
- 移除成员 → 二次确认弹窗
- 角色修改 → 下拉选择 → 实时更新
- 所有权转让 → 选择成员 → 输入密码 → 确认
- 空状态：无成员时的空列表提示

### ProjectRolesModal（项目角色管理）
- 角色列表
- 创建角色 → 名称 → 选择权限 → 确认
- 编辑角色 → 修改名称/权限
- 删除角色 → 二次确认
- 权限 checkbox 组：按类别分组（项目/文件/CAD/版本）

## 种子数据要求

- 每种系统角色至少一个用户（ADMIN/USER_MANAGER/FONT_MANAGER/USER）
- 每种项目角色至少一个成员（OWNER/ADMIN/MEMBER/EDITOR/VIEWER）
- 预建项目含多角色成员，用于权限交叉验证
- storageState：至少保存 ADMIN 和 USER 的登录态
