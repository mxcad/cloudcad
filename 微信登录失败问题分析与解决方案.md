# 微信登录失败问题分析与解决方案

## 问题描述

用户在使用微信登录时遇到 "账号已被禁用" 的错误，且前端没有明显的错误提示，导致微信登录无法正常使用。

## 根本原因分析

### 1. 后端验证逻辑

- **文件**：`packages/backend/src/auth/auth-facade.service.ts:457-459`
- **问题**：在微信登录过程中，后端会检查用户状态，如果用户状态不是 `ACTIVE`，则抛出 "账号已被禁用" 错误
- **代码**：
  ```typescript
  if (user.status !== 'ACTIVE') {
    throw new UnauthorizedException('账号已被禁用');
  }
  ```

### 2. 前端错误处理

- **文件**：`packages/frontend/src/contexts/AuthContext.tsx:363-365`
- **问题**：前端在处理微信登录结果时，对于错误情况只是在控制台打印错误，没有显示给用户
- **代码**：
  ```typescript
  if (result.error) {
    console.error('微信登录失败:', result.error);
    // 可以在这里添加错误提示，例如 showToast
  }
  ```

### 3. 用户状态管理

- **文件**：`packages/backend/prisma/schema.prisma:27`
- **问题**：用户模型包含 `status` 字段，默认值为 `INACTIVE`，当用户状态变为 `SUSPENDED` 时会导致登录失败
- **代码**：
  ```prisma
  status    UserStatus @default(INACTIVE)
  ```

## 解决方案

### 前端改进

#### 1. 改进 AuthContext 中的错误处理

**修改文件**：`packages/frontend/src/contexts/AuthContext.tsx`

**改进内容**：

- 添加错误状态管理
- 在微信登录失败时显示错误提示
- 确保错误信息能够传递到登录页面

**代码示例**：

```typescript
// 在 AuthContextType 中添加错误状态
interface AuthContextType {
  // 现有字段...
  error: string | null;
  setError: (error: string | null) => void;
}

// 在 AuthProvider 中添加错误状态管理
const [error, setError] = useState<string | null>(null);

// 修改微信登录结果处理
if (result.error) {
  console.error('微信登录失败:', result.error);
  setError(result.error);
  // 可以使用 toast 或其他方式显示错误
}

// 在 value 中添加 error 和 setError
const value = useMemo<AuthContextType>(
  () => ({
    // 现有字段...
    error,
    setError,
  }),
  [
    // 现有依赖...
    error,
  ]
);
```

#### 2. 改进 Login 组件中的微信登录错误处理

**修改文件**：`packages/frontend/src/pages/Login.tsx`

**改进内容**：

- 在微信登录失败时显示错误信息
- 确保错误信息能够清晰地展示给用户

**代码示例**：

```typescript
// 在 Login 组件中使用 error 状态
const { loginWithWechat, error, setError } = useAuth();

// 修改微信登录按钮的点击处理
<button
  type="button"
  onClick={async () => {
    try {
      setError(null);
      await loginWithWechat();
    } catch (err) {
      const errorMessage = (
        err as Error & {
          response?: { data?: { message?: string } };
        }
      ).response?.data?.message ||
        (err as Error).message ||
        '微信登录失败';
      setError(errorMessage);
    }
  }}
  className="wechat-login-button"
>
  <MessageCircle size={20} />
  <span>微信登录</span>
</button>

// 在消息提示区域显示错误
{error && (
  <div className="alert alert-error">
    <AlertCircle size={18} className="alert-icon" />
    <span>{error}</span>
  </div>
)}
```

### 后端改进

#### 1. 改进错误信息的清晰度

**修改文件**：`packages/backend/src/auth/auth-facade.service.ts`

**改进内容**：

- 提供更具体的错误信息，区分不同类型的禁用状态
- 确保错误信息能够清晰地传递给前端

**代码示例**：

```typescript
if (user.status === 'SUSPENDED') {
  throw new UnauthorizedException('账号已被暂停使用');
} else if (user.status === 'INACTIVE') {
  throw new UnauthorizedException('账号尚未激活');
} else if (user.status !== 'ACTIVE') {
  throw new UnauthorizedException('账号状态异常');
}
```

#### 2. 确保用户注销后状态正确

**修改文件**：`packages/backend/src/auth/services/auth-token.service.ts`

**改进内容**：

- 确保用户注销操作不会影响用户状态
- 保持用户状态的一致性

### 数据库管理

#### 1. 检查用户状态

**操作**：

- 检查数据库中用户的状态
- 确保用户状态与实际情况一致
- 对于被禁用的用户，提供管理员界面进行状态管理

**SQL 查询示例**：

```sql
-- 查看所有非活跃状态的用户
SELECT id, username, email, status FROM users WHERE status != 'ACTIVE';

-- 将用户状态恢复为活跃
UPDATE users SET status = 'ACTIVE' WHERE id = '用户ID';
```

- 微信登录功能恢复正常使用

## 总结

##

微信登录失败的根本原因是用户状态异常导致的后端验证失败，而前端缺乏有效的错误提示机制。通过改进前端错误处理、后端错误信息清晰度以及数据库状态管理，可以有效解决这一问题，提升用户体验。
