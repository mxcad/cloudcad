# 系统架构（Shared）

## 概述

CloudCAD 采用 monorepo 架构，基于 NestJS + React 构建，专为 B2B 私有部署设计。系统采用本地文件系统存储，支持智能目录分配、分片上传、CAD 图纸转换等核心功能。

## 核心架构模式

- **前后端分离**：NestJS 后端 + React 前端
- **Monorepo 结构**：使用 pnpm workspace 管理多包
- **RBAC 权限控制**：基于角色的细粒度权限系统
- **本地文件存储**：基于文件系统的存储架构，支持智能目录分配
- **多级缓存**：权限缓存、角色缓存、Redis 缓存
- **分片上传**：支持大文件分片上传和断点续传

## 技术栈

**后端**：NestJS 11.0.1, TypeScript 5.7.3, Express 5.2.1, Prisma 7.1.0, PostgreSQL 15+, Redis 5.10.0

**前端**：React 19.2.1, TypeScript ~5.8.2, Vite 6.2.0, React Router DOM 7.10.1, Tailwind CSS 4.1.18

## 核心模块

| 模块 | 职责 |
|------|------|
| 认证系统 | JWT 双 Token + Session 支持 |
| 权限系统 | RBAC 权限控制 + 多级缓存 |
| 文件系统 | 统一树形结构管理 |
| 存储管理 | 本地存储 + 智能目录分配 |
| MxCAD 转换 | CAD 图纸上传和转换 |
| 图库管理 | 图纸库和图块库管理 |
| 审计日志 | 操作审计和追踪 |

## 数据流

```
用户请求 → JWT 验证 → 权限检查 → 业务逻辑 → 数据库操作 → 存储操作 → 响应返回
```

## 存储架构

```
filesData/
├── YYYYMM/          # 年月主目录
│   └── {nodeId}/    # 节点目录
├── YYYYMM_1/        # 子目录（主目录满时创建）
└── YYYYMM_2/
```

## 关键特性

- 文件去重（SHA-256 哈希）
- 软删除 + 延迟物理删除
- 并发控制（文件锁 + 限流）
- 路径安全验证
- 自动清理过期文件
