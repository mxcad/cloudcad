# 部署引导配置方案

> 创建日期：2026-03-14
> 最后更新：2026-03-16
> 状态：设计中

## 1. 概述

为用户提供简洁的首次部署引导，通过命令行交互完成必要配置。

**核心目标**：
- 命令行交互，简单直接
- 回车使用默认值，快速完成
- 密码强度检查，给出警告但不阻止

## 2. 与运行时配置的区别

| 特性 | 部署引导配置 | 运行时配置中心 |
|------|-------------|---------------|
| **存储位置** | .env 文件 | 数据库 + Redis |
| **修改生效** | 需重启服务 | 立即生效 |
| **配置内容** | 数据库连接、端口、密码等基础设施配置 | 功能开关、业务参数 |
| **安全性** | 敏感信息，不应暴露给前端 | 非敏感，部分可公开 |

## 3. 启动流程

```
start.bat / start.sh
    │
    ▼
检测 packages/backend/.env 是否存在
    │
    ├─ 存在 ────────────────────→ 正常启动
    │
    └─ 不存在 ──→ 运行引导配置
                        │
                        ▼
                  命令行交互（输入配置）
                        │
                        ▼
                  写入 .env 文件
                        │
                        ▼
                  执行部署 cli.js deploy
                        │
                        ▼
                  部署完成 → 显示访问地址
```

## 5. 交互规则

### 5.1 密码输入

- 输入时不显示明文
- 管理员密码留空则自动生成（12位随机字符）
- **密码强度检查**：

```
管理员密码 [自动生成]: 123456

⚠️  警告: 密码过于简单，存在安全风险
   建议使用包含大小写字母、数字、特殊字符的密码

确认使用此密码? [y/N]: _
```

- 强度检查规则：
  - 长度 < 8 位 → 简单
  - 只有数字或只有字母 → 简单
  - 其他 → 复杂

### 5.2 配置确认

- 显示配置摘要
- 用户确认后才写入 .env 文件
- 输入 `n` 可重新配置

## 6. 配置项清单

### 6.1 引导输入项

| 配置项 | 环境变量 | 默认值 |
|--------|----------|--------|
| 数据库密码 | DB_PASSWORD | 内置默认密码 |
| 管理员密码 | INITIAL_ADMIN_PASSWORD | 自动生成 12 位随机密码 |

**说明**：所有配置都有默认值，用户可一路回车快速完成配置。

### 6.2 其他默认配置

| 配置项 | 环境变量 | 默认值 |
|--------|----------|--------|
| 服务端口 | PORT | 3001 |
| 数据库主机 | DB_HOST | localhost |
| 数据库端口 | DB_PORT | 5432 |
| 数据库名称 | DB_DATABASE | cloudcad |
| 数据库用户 | DB_USERNAME | postgres |
| Redis 主机 | REDIS_HOST | localhost |
| Redis 端口 | REDIS_PORT | 6379 |
| JWT 密钥 | JWT_SECRET | 自动生成 |

## 7. 实现位置

引导配置在 `runtime/scripts/cli.js` 的 `bootstrap()` 函数中实现：

```javascript
async function bootstrap() {
  // 1. 设置离线环境（包含 copyEnvExampleToEnv，复制 .env.example → .env）
  const success = await setupOffline({ silent: true, deployBackendOnly: isDeployMode });
  
  // 2. 检测 .env 是否刚创建（首次部署）
  const envPath = path.join(PROJECT_ROOT, 'packages', 'backend', '.env');
  const envJustCreated = /* 判断是否刚复制 */;
  
  if (envJustCreated) {
    // 3. 读取 .env 提取默认值
    // 4. 运行引导配置，让用户修改
    await runSetupWizard();
  }
  
  // ... 后续逻辑
}
```

**流程说明**：
1. `setupOffline()` 内部调用 `copyEnvExampleToEnv()` 复制 `.env.example` → `.env`
2. 读取 `.env` 文件，提取 `DB_PASSWORD` 和 `INITIAL_ADMIN_PASSWORD` 的默认值
3. 引导用户输入，回车保留默认值
4. 用户确认后，写入修改后的值到 `.env`

## 8. 后续修改配置

部署完成后，用户可通过以下方式修改配置：

1. 直接编辑 `packages/backend/.env` 文件，然后重启服务
2. 重新运行 `setup.bat` / `setup.sh`
3. 运行时配置中心（非敏感配置）
