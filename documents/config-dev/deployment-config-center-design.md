# 部署配置中心设计方案

> 创建日期：2026-03-14
> 状态：设计中

## 1. 概述

部署配置中心是一个独立的服务，用于管理项目的部署配置（.env 文件）。

**核心特点**：
- 独立服务，不受主项目影响
- 可视化管理配置项
- 配置验证，避免错误配置
- 一键重启，方便小白用户

## 2. 与其他配置的区别

| 特性 | 部署引导 | 部署配置中心 | 运行时配置中心 |
|------|---------|-------------|---------------|
| **时机** | 首次部署 | 随时可修改 | 随时可修改 |
| **存储** | .env 文件 | .env 文件 | 数据库 |
| **生效** | 需重启 | 需重启 | 立即生效 |
| **入口** | 引导页面 | 独立服务 (3002) | 主服务后台 |

## 3. 架构设计

### 3.1 独立服务

| 特性 | 说明 |
|------|------|
| 独立端口 | 3002（主服务 3001） |
| 独立运行 | 不依赖数据库、Redis |
| 随主服务启动 | 由 PM2 统一管理 |

```
┌─────────────────────────────────────────────────────┐
│                    PM2 进程管理                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│   ┌─────────────────┐   ┌─────────────────┐        │
│   │   主服务         │   │  配置服务        │        │
│   │   Port: 3001    │   │  Port: 3002     │        │
│   │                 │   │                 │        │
│   │  - 业务 API     │   │  - 配置管理     │        │
│   │  - 数据库连接   │   │  - 重启控制     │        │
│   │  - Redis 连接   │   │  - 连接测试     │        │
│   └─────────────────┘   └─────────────────┘        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 3.2 与主服务的关系

| 场景 | 主服务状态 | 配置服务状态 |
|------|-----------|-------------|
| 正常运行 | 运行 | 运行 |
| 主服务崩溃 | 停止 | 运行（可修复配置） |
| 修改配置 | 运行 → 重启 | 运行 |

## 4. 访问控制

### 4.1 认证方式

| 方式 | 说明 |
|------|------|
| 管理员密码 | 使用部署引导时设置的 ADMIN_PASSWORD |
| 独立登录页 | 配置服务有独立的登录页面 |
| Session 管理 | 登录后创建 Session，有效期 30 分钟 |

### 4.2 安全措施

| 措施 | 说明 |
|------|------|
| 登录失败锁定 | 连续失败 5 次后锁定 15 分钟 |
| Session 超时 | 30 分钟无操作自动登出 |
| 操作日志 | 记录所有配置修改操作 |
| 敏感信息保护 | 密码类字段不显示原文 |

## 5. 配置项管理

### 5.1 配置项分类

| 分类 | 配置项 | 后台可见 | 可修改 | 显示方式 |
|------|--------|---------|--------|----------|
| **服务** | PORT | ✓ | ✓ | 正常显示 |
| **数据库** | DB_HOST, DB_PORT, DB_NAME, DB_USER | ✓ | ✓ | 正常显示 |
| **数据库** | DB_PASSWORD | ✓ | ✓ | 显示「已设置/未设置」 |
| **Redis** | REDIS_HOST, REDIS_PORT | ✓ | ✓ | 正常显示 |
| **Redis** | REDIS_PASSWORD | ✓ | ✓ | 显示「已设置/未设置」 |
| **安全** | JWT_SECRET | ✓ | ✓ | 显示「已设置」，支持重新生成 |
| **邮件** | MAIL_ENABLED, MAIL_HOST, MAIL_PORT, MAIL_USER | ✓ | ✓ | 正常显示 |
| **邮件** | MAIL_PASSWORD | ✓ | ✓ | 显示「已设置/未设置」 |

### 5.2 敏感字段处理

**密码类字段**：
- 不显示原文
- 显示状态：「已设置」或「未设置」
- 提供「修改」按钮，输入新值
- 提供「清空」按钮（可选）

**JWT 密钥**：
- 显示状态：「已设置」
- 提供「重新生成」按钮
- 重新生成后需重启服务

## 6. 配置验证

### 6.1 保存前验证

| 配置项 | 验证规则 | 失败提示 |
|--------|----------|----------|
| PORT | 1-65535 范围内的数字 | 端口必须在 1-65535 之间 |
| DB_* | 实际测试数据库连接 | 数据库连接失败：{错误信息} |
| REDIS_* | 实际测试 Redis 连接 | Redis 连接失败：{错误信息} |
| MAIL_* | 实际测试 SMTP 连接（如启用） | SMTP 连接失败：{错误信息} |

### 6.2 验证流程

```
用户修改配置
    │
    ▼
前端基础验证（格式、必填项）
    │
    ▼
后端连接测试
    │
    ├─ 测试通过 ──> 允许保存
    │
    └─ 测试失败 ──> 显示错误信息，阻止保存
```

### 6.3 错误提示示例

```
❌ 数据库连接失败
   错误：Connection refused (host: localhost, port: 5432)
   请检查数据库服务是否启动
```

## 7. 修改生效流程

### 7.1 保存后提示

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   ✅ 配置已保存！                                    │
│                                                     │
│   [立即重启服务]    [稍后手动重启]                    │
│                                                     │
│   提示：重启期间服务将短暂不可用（约 5-10 秒）         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 7.2 重启流程

**一键重启**：
1. 点击「立即重启服务」
2. 配置服务调用 PM2 API 重启主服务
3. 显示重启进度
4. 重启完成后显示成功提示

**手动重启**：
```
请执行以下命令重启服务：

Windows: stop.bat && start.bat
Linux:   ./stop.sh && ./start.sh
```

## 8. API 设计

### 8.1 认证相关

| API | 方法 | 说明 |
|-----|------|------|
| `/config/auth/login` | POST | 管理员登录 |
| `/config/auth/logout` | POST | 登出 |
| `/config/auth/status` | GET | 获取登录状态 |

### 8.2 配置管理

| API | 方法 | 说明 |
|-----|------|------|
| `/config/settings` | GET | 获取所有配置项 |
| `/config/settings` | PUT | 更新配置项 |
| `/config/settings/:key` | GET | 获取单个配置项 |
| `/config/settings/:key` | PUT | 更新单个配置项 |

### 8.3 连接测试

| API | 方法 | 说明 |
|-----|------|------|
| `/config/test/database` | POST | 测试数据库连接 |
| `/config/test/redis` | POST | 测试 Redis 连接 |
| `/config/test/smtp` | POST | 测试 SMTP 连接 |

### 8.4 服务控制

| API | 方法 | 说明 |
|-----|------|------|
| `/config/service/restart` | POST | 重启主服务 |
| `/config/service/status` | GET | 获取主服务状态 |
